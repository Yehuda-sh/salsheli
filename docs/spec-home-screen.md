# 📱 מסמך אפיון — מסך הבית (Dashboard)

## סקירה כללית

מסך הבית הוא הדף הראשון שהמשתמש רואה אחרי התחברות. הוא מציג תמונת מצב של הקניות, המזווה וההוצאות — הכל בעמוד אחד.

**קבצים:**
- `lib/screens/home/dashboard/home_dashboard_screen.dart` (1,239 שורות)
- 4 widgets משניים ב-`widgets/` (1,817 שורות)
- **סה"כ: ~3,056 שורות**

## 📦 רכיבים בפירוט

### 1. 🔌 באנר אין חיבור (OfflineBanner)
- **מיקום:** בראש הדף, מעל הכל
- **מתי מופיע:** כשהטלפון לא מחובר לאינטרנט (WiFi/סלולר)
- **עיצוב:** פס אדום עם אייקון WiFi מנותק + "אין חיבור לאינטרנט"
- **התנהגות:** נעלם אוטומטית כשהחיבור חוזר
- **מקור מידע:** `connectivity_plus` package — מאזין לשינויי רשת

### 2. ⚠️ באנר שגיאה (Error Banner)
- **מתי מופיע:** כשטעינת הרשימות מ-Firebase נכשלה
- **עיצוב:** רקע אדום בהיר + אייקון שגיאה + טקסט השגיאה
- **כפתור:** "נסה שוב" — מפעיל רענון של כל הנתונים
- **מקור מידע:** `ShoppingListsProvider.hasError` + `errorMessage`

### 3. 🛒 באנר קנייה פעילה (ActiveShopperBanner)
- **מתי מופיע:** בשני מקרים:
  - **א.** אני באמצע קנייה (יצאתי מהאפליקציה וחזרתי) → "להמשיך לקנות?"
  - **ב.** מישהו אחר מהבית קונה עכשיו → "רונית קונה עכשיו ב'סופר השבוע'!"
- **כפתור:** לחיצה → מעבר למסך קנייה פעילה
- **עיצוב:** כרטיס ירוק עם אנימציית כניסה + אייקון עגלה
- **מקור מידע:** `ShoppingListsProvider.lists` — מחפש רשימות עם `activeShopper`

### 4. 📩 באנר הזמנות ממתינות (PendingInvitesBanner)
- **מתי מופיע:** כשמישהו הזמין אותי לבית/רשימה ועדיין לא טיפלתי
- **עיצוב:** כרטיס כתום עם shimmer על אייקון המעטפה
- **כפתור:** לחיצה → מעבר למסך הזמנות ממתינות (`/pending-invites`)
- **מקור מידע:** `PendingInvitesService` — מושך הזמנות לפי אימייל המשתמש

### 5. 👤 Header (ברכה + אווטר + התראות)

**צד ימין — אווטר:**
- עיגול עם ראשי תיבות של השם (למשל "י" ליהודה, "אכ" לאבי כהן)
- **לחיצה:** פותח Bottom Sheet עם פרטי המשתמש:
  - אווטר גדול
  - שם מלא
  - שם הבית (עם אייקון בית 🏠)
  - סטטוס "מחובר" (נקודה ירוקה)
  - כתובת אימייל

**מרכז — ברכה דינמית:**
- משתנה לפי שעה:
  - לפני 12:00 → "☀️ בוקר טוב, יהודה!"
  - 12:00-17:00 → "🌤️ צהריים טובים, יהודה!"
  - 17:00-21:00 → "🌅 ערב טוב, יהודה!"
  - אחרי 21:00 → "🌙 לילה טוב, יהודה!"
- **שורה שנייה (כתובית):**
  - אם יש רשימות פעילות → "3 רשימות פעילות"
  - אם אין → שם הבית (למשל "הבית של כהן")

**צד שמאל — פעמון התראות:**
- אייקון פעמון עם badge (מספר) אם יש התראות שלא נקראו
- **לחיצה:** מעבר למסך התראות (`/notifications`)
- **מקור מידע:** `NotificationsService.watchUnreadCount()` — stream בזמן אמת

### 6. ⚡ Quick Actions (3 כפתורי קיצור)

שלושה כפתורים בשורה אחת:

| כפתור | אייקון | צבע | יעד |
|--------|--------|------|------|
| סופרמרקט | 🛒 | ירוק (accent) | `/create-list` — יצירת רשימה חדשה |
| קבלות | 🧾 | ירוק (success) | `/receipts` — היסטוריית קניות |
| התראות | 🔔 | סגול (tertiary) | `/notifications` — מסך התראות |

- **עיצוב:** כרטיסים שקופים עם אייקון בעיגול + כיתוב
- **אינטראקציה:** Haptic feedback בלחיצה, Material ripple effect

### 7. 📊 סיכום חודשי (Monthly Summary)

- **מתי מופיע:** רק אם יש לפחות קבלה אחת מהחודש הנוכחי
- **עיצוב:** כרטיס גרדיאנט (primary → secondary) עם 3 עמודות:

| עמודה | אייקון | מה מציג | דוגמה |
|--------|--------|---------|--------|
| הוצאות | 🧾 | סכום כולל החודש | ₪850 |
| קניות | 🛍️ | מספר קניות החודש | 4 |
| ממוצע | 📉 | ממוצע לקנייה | ₪212 |

- **מקור מידע:** `ReceiptProvider.receipts` — מסנן לפי חודש נוכחי

### 8. 💡 הצעות להיום (SuggestionsTodayCard)

- **מה זה:** קרוסלה אופקית של "פתקיות דביקות" (Sticky Notes) עם מוצרים שכדאי לקנות
- **מקור ההצעות:** `SuggestionsProvider` — מחשב לפי:
  - מוצרים עם מלאי נמוך במזווה
  - מוצרים שנגמרים בתדירות גבוהה
  - עד 3 הצעות מוצגות
- **כל פתקית מכילה:**
  - אימוגי + שם מוצר
  - כמות נוכחית במזווה
  - כפתור ➕ להוספה לרשימה פעילה
  - כפתור ✕ לדחיית ההצעה
- **עיצוב:** פתקיות צבעוניות עם סיבוב קל, צלליות, אפקט "הרמה" בלחיצה
- **מתי לא מופיע:** אם אין הצעות או אם המזווה ריק

### 9. 🛒 רשימות פעילות (Active Lists)

**אם אין רשימות:**
- כרטיס ריק מזמין עם אייקון + "אין רשימות פעילות" + "הוסף רשימה חדשה"
- **לחיצה:** מעבר ליצירת רשימה (`/create-list`)

**אם יש רשימות — כל כרטיס מכיל:**

```
┌─────────────────────────────────────┐
│🟢│ 🛒  סופר השבוע                   │
│  │      ████████░░░░  3/10          │
│  │      🛍️ נותרו 7 פריטים    ←     │
└─────────────────────────────────────┘
```

| אלמנט | תיאור |
|--------|--------|
| פס צד צבעוני | צבע לפי סוג הרשימה (סופר=ירוק, בית מרקחת=כחול...). הושלם=ירוק |
| אימוגי בעיגול | אימוגי לפי סוג (🛒 סופר, 💊 בית מרקחת, 🥩 קצביה...) |
| שם הרשימה | בולד, שורה אחת |
| Progress bar | פס התקדמות (נסמנו/סה"כ) |
| סטטוס | "נותרו X פריטים" או "✅ הושלם!" |
| ספירה | "3/10" בפינה |
| חץ | חץ לניווט (RTL aware — שמאלה בעברית) |

- **מיון:** לפי תאריך עדכון אחרון (חדש קודם)
- **לחיצה:** מעבר לפרטי הרשימה (`/list-details`)
- **Haptic feedback:** בכל לחיצה

### 10. 📜 היסטוריה (History)

- **מציג:** 2 קבלות אחרונות בלבד
- **כותרת:** "היסטוריה" + כפתור "ראה הכל" (אם יש יותר מ-2)
- **כל קבלה מציגה:**
  - אייקון קבלה בריבוע צבעוני
  - שם החנות
  - תאריך יחסי ("היום", "אתמול", "לפני 3 ימים", "12/3/2026")
  - סכום כולל (₪320) — או מספר פריטים אם אין סכום
- **"ראה הכל":** מעבר למסך היסטוריה מלאה (`ShoppingHistoryScreen`)
- **לחיצה על קבלה:** מעבר להיסטוריה עם הקבלה הספציפית פתוחה

### 11. ➕ FAB (כפתור צף — "רשימה חדשה")

- **מיקום:** פינה שמאלית תחתונה (RTL → שמאל)
- **עיצוב:** כפתור ירוק מורחב עם אייקון + וטקסט "רשימה חדשה"
- **אנימציה:** אפקט scale elastic בטעינה ראשונה (קופץ פנימה)
- **לחיצה:** מעבר ליצירת רשימה חדשה (`/create-list`)

## 🔄 תזרים מידע (Data Flow)

```
Firebase Firestore
    │
    ├── shopping_lists/{id}
    │       ↓
    │   ShoppingListsProvider (ChangeNotifier)
    │       ↓
    │   ├── activeLists → כרטיסי רשימות
    │   ├── hasError → באנר שגיאה
    │   └── lists[].activeShopper → באנר קנייה פעילה
    │
    ├── receipts/{id}
    │       ↓
    │   ReceiptProvider (ChangeNotifier)
    │       ↓
    │   ├── receipts → היסטוריה + סיכום חודשי
    │   └── receipts.isEmpty → הסתר סקשן
    │
    ├── users/{uid}
    │       ↓
    │   UserContext (ChangeNotifier)
    │       ↓
    │   ├── displayName → ברכה
    │   ├── householdId → שם הבית
    │   ├── userId → queries
    │   └── userEmail → פרטי פרופיל
    │
    ├── users/{uid}/inventory
    │   OR households/{id}/inventory
    │       ↓
    │   SuggestionsProvider (ChangeNotifier)
    │       ↓
    │   └── suggestions → כרטיס הצעות
    │
    ├── notifications/{uid}
    │       ↓
    │   NotificationsService (stream)
    │       ↓
    │   └── unreadCount → badge על הפעמון
    │
    └── pending_invites/{id}
            ↓
        PendingInvitesService (one-time fetch)
            ↓
        └── invites.isNotEmpty → באנר הזמנות
```

### Providers ב-Dashboard:

| Provider | מה מספק | סוג |
|----------|---------|------|
| `ShoppingListsProvider` | רשימות פעילות, שגיאות | watch (real-time) |
| `ReceiptProvider` | קבלות, היסטוריה | watch |
| `UserContext` | שם, אימייל, בית | watch |
| `SuggestionsProvider` | הצעות חכמות | watch |
| `NotificationsService` | ספירת התראות | stream |
| `PendingInvitesService` | הזמנות ממתינות | one-time |

## 🎨 עיצוב ואנימציות

### רקע
- `NotebookBackground` — רקע מחברת עם שורות דקות

### אנימציות כניסה (חד-פעמי, טעינה ראשונה)
- כל סקשן נכנס עם **fade + slide up** (400ms, delay 50ms × index)
- FAB נכנס עם **elastic scale** (500ms, delay 300ms)
- באנרים עם אנימציית כניסה ייחודית (flutter_animate)

### Pull-to-Refresh
- משיכה למטה → רענון כל הנתונים (רשימות + קבלות + הצעות)
- Haptic feedback: medium בתחילה, light בסיום
- מינימום 300ms (להרגשת refresh)

### צבעים דינמיים
- כל הצבעים מגיעים מ-`Theme.colorScheme` (תומך Dark Mode)
- צבעי רשימות מ-`ListTypes.getColor()` (לפי סוג חנות)
- צבע accent מ-`AppBrand` extension

### RTL Support
- חיצים מתהפכים (chevron_left בעברית)
- פס צד צבעוני בצד ימין (RTL)
- כיוון טקסט אוטומטי
